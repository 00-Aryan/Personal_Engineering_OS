# Problem Definition

## Core Problem

Aryan cannot simultaneously maintain:

- Academic focus for IIT Madras final year
- Project development for content creation pipeline work and data science
  projects
- Startup ideation

ProjectOS exists because current AI tools are reactive and operate only
after being asked. Aryan needs a proactive system that runs continuously
without him.

## Specific Pain Points

ProjectOS must address these source-defined pain points:

- No continuous code quality enforcement
- No architecture challenge before building wrong
- Tests get skipped because they are painful, which then breaks production
- Context loss across sessions, including loss of why decisions were made
- Aryan cannot hold multiple engineering mindsets simultaneously
- Multiple tasks at once cause hallucination or shallow output
- One blocked permission halts the entire workflow

## Required Problem Response

The system must reduce the need for Aryan to context-switch between
engineering roles by assigning engineering responsibilities to multiple
agents and by using a Clone meta-agent to supervise them.

The system must run in the background as a daemon while Aryan studies.
